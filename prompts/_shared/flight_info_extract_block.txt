## 航班信息识别核心规则

### 航班号识别（高风险，必须逐字符核对）
航班号中的每一位数字都必须独立核对，绝对禁止根据上下文推断或"补全"任何一位数字。以下是高频混淆场景，必须特别注意：
- 末位数字混淆：QZ646 ≠ QZ648，MU5183 ≠ MU5188，CA1234 ≠ CA1235，数字 6/8、3/8、1/7 极易混淆
- 字母混淆：字母 O 与数字 0、字母 I 与数字 1、字母 B 与数字 8
- 若同一材料中出现多个相似航班号（如 QZ646 和 QZ648），必须全部列入 all_flights_found，绝对禁止合并或只保留其中一个
- 若图片中航班号被 logo、图标、折叠区域部分遮挡，必须在 extraction_notes 中注明"航班号存在遮挡，已尽力识别为 XXX，建议人工核查"

### 行程单与登机牌分开识别
行程单/电子客票上的航班号是"原定航班"，登机牌上的航班号是"实际乘坐航班"。若两者不同，说明发生了改签，必须将两个航班号都列入 all_flights_found，绝对禁止只保留其中一个。行程单上的原定航班即使已取消，也必须列出。

### 行程变动分区截图（高频场景，必须识别）
航旅APP（如携程、飞猪、去哪儿、Traveloka等）的行程变动通知截图中，常见"原航班/原定航班"与"行程变动后的新航班/新安排"两个分区并排显示。**两个分区的航班号必须全部列入 all_flights_found**，绝对禁止只提取其中一个分区的航班号。典型标志词：「原航班」「行程变动后的新航班」「新安排」「已变更」「原定」「更改为」等。

- **关键警告**：两个分区的航班号看起来相似时（如同为QZ64X），绝对禁止将两个分区的航班号归并为同一个。必须对每个分区的航班号独立逐字符核对后分别列出，即使末位数字只差1位（如QZ646 vs QZ648）也必须如实记录两个不同的航班号。

### 机场三字码
延误证明、行程单、登机牌上的机场三字码（如 SVO、HGH、PVG、HKT 等）必须提取，用于时区换算和延误时长计算。

### 时间格式
统一输出 "YYYY-MM-DD HH:MM"

## 第一步：枚举所有航班（必须先完成此步骤）

将材料中出现的**每一个航班号**（无论来自行程单、登机牌、改签通知、保护航班提示、飞常准截图等）全部列出：

{
  "all_flights_found": [
    {
      "flight_no": "航班号，逐字核对数字",
      "date": "YYYY-MM-DD 或 YYYY-MM-DD HH:MM/unknown",
      "dep_iata": "出发机场三字码/unknown",
      "arr_iata": "到达机场三字码/unknown",
      "source": "来源材料（行程单/登机牌/改签通知/保护航班提示/飞常准截图等）",
      "role_hint": "材料中对该航班的描述原文（如'原航班取消''保护航班''改签''第1程''第2程'等，直接引用原文）"
    }
  ]
}

## 第二步：角色判定规则

根据 all_flights_found 中的信息，按以下规则判定每个航班的角色：

**原航班**（original）：
- 行程单/电子客票中列出的原定航班
- 多程行程中每一程的原定航班均需识别
- 若某程原航班取消/延误，该航班仍为原航班

**改签/替代航班**（alternate）：
- 原航班取消/延误后，承运人或客户重新购买的替代航班
- 登机牌上的航班若与行程单原航班不同，通常为改签航班
- 改签通知/短信中出现的新航班号

**保护航班**（protection）：
- 材料中明确出现"保护航班""protection flight"字样的航班
- 通常由航空公司在原航班取消时主动安排，客户可选择乘坐或自行改签

**注意**：
- 登机牌上的航班号 ≠ 一定是改签航班，需结合行程单对比判断
- 若行程单有两程，第一程正常到达、第二程取消，则第二程原航班为延误险的触发航班
- 若客户实际乘坐的航班（登机牌）与行程单原航班一致，则无改签
- **行程变动截图的角色判定（高频场景）**：航旅APP行程变动截图中，「原航班」区块的航班 = original（原航班），「行程变动后的新航班」/「新安排」区块的航班 = alternate（替代航班）。绝对禁止将「新航班」区块的航班号填入 flight_no（触发延误险的原航班）。flight_no 必须填「原航班」区块的航班号。

## 第三步：保护航班时间合理性校验

若存在保护航班，需校验：**客户实际乘坐的替代/改签航班起飞时间，必须晚于保护航班起飞时间**（客户放弃保护航班、自行选择更晚的替代方案，才符合"最早便利替代"原则）。

- 若替代航班时间 < 保护航班时间：`protection_check.result = "异常"` 并说明原因
- 若替代航班时间 ≥ 保护航班时间：`protection_check.result = "正常"`
- 若保护航班时间 unknown（材料未载明）：先记录保护航班号，后续由飞常准查询补全
