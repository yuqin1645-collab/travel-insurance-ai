## 任务背景

该案件已确认包含航空公司出具的PIR报告（行李运输事故报告），但主扫描流程未能从中提取行李签收时间。现在请你**只关注与行李签收/交付相关的时间信息**，忽略其他内容。

## 重点关注的文件类型

1. **PIR报告（Property Irregularity Report）**：行李运输事故单，通常包含：
   - 行李预计送达日期/时间
   - 行李追踪状态（含"Delivered"/"Consegnato"/"Entregado"等已交付状态的日期）
   - 航司承诺的行李送达时间

2. **行李签收单/交付单**：
   - 航司/机场/快递出具的行李交付凭证
   - 含签收日期和时间的文件
   - 有签收人签字栏的交付单

3. **行李追踪记录**：
   - 航司官网或APP的行李追踪截图
   - 显示"已送达""已交付"状态的页面

## 多语言识别指引

PIR报告可能以任何语言出具，请重点识别以下关键词对应的日期时间：

| 语言 | 送达/交付/签收关键词 |
|------|---------------------|
| 中文 | 送达、交付、签收、领取、收到行李 |
| 英文 | Delivery, Delivered, Received, Claimed, Handed over |
| 意大利语 | Consegnato, Data di consegna, Ritiro |
| 西班牙语 | Entregado, Entrega, Fecha de entrega, Recogida |
| 葡萄牙语 | Entregue, Entrega, Data de entrega, Recebido |
| 法语 | Livré, Livraison, Délivré, Date de livraison |
| 德语 | Zugestellt, Ausgabe, Auslieferung |

## 输出 JSON（仅包含以下字段）

```json
{
  "baggage_receipt_time": "YYYY-MM-DD HH:MM（最晚的行李签收时间）/unknown",
  "receipt_times": ["YYYY-MM-DD HH:MM（所有识别到的签收时间）"],
  "delay_hours": number（如果能从 receipt_time - flight_actual_arrival_time 推算）/unknown,
  "confidence": "high/medium/low",
  "source_file": "识别到签收时间的文件名/unknown",
  "notes": "提取依据的简要说明/unknown"
}
```

## 重要规则

1. **只关注行李签收时间**，不要做其他字段提取
2. **日期格式统一**为 YYYY-MM-DD HH:MM，如果只有日期没有具体时间，使用 YYYY-MM-DD
3. **多份签收证明**：列出全部到 receipt_times，最晚的写入 baggage_receipt_time
4. **置信度判断**：
   - high：文件明确显示"已交付/已签收"状态 + 具体日期时间
   - medium：有交付日期但缺少具体时间，或文件类型略有模糊
   - low：仅有推测或模糊的日期信息
5. **如果完全找不到行李签收时间信息**，所有时间字段输出 unknown，confidence 输出 low
