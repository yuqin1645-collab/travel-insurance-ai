# 架构文档一致性检查清单

## 1. 使用说明

本清单用于检查以下文档是否与当前代码实现一致：

- `docs/系统架构总结.md`
- `docs/架构决策记录-ADR-001-模块化与命名空间隔离.md`

建议在以下场景执行一次：

- 新增或下线 `claim_type`
- 调整模块路由、Prompt 命名空间、条款映射
- 修改审核结论口径（补件/人工复核/拒赔/通过）
- 进行重大重构或上线前巡检

## 2. 代码与文档对齐项

### 2.1 模块与路由

- [ ] `app/modules/registry.py` 中注册的模块列表，与架构文档“当前已启用模块”一致
- [ ] 入口路由中 `claim_type` 识别规则，与文档描述一致
- [ ] 审核结果落盘目录按 `claim_type` 隔离，与文档描述一致

### 2.2 Prompt 命名空间

- [ ] `prompts/<claim_type>/` 目录存在且与文档一致
- [ ] 代码调用 `prompt_loader.format(..., namespace=...)` 的命名空间与模块定义一致
- [ ] 不存在跨险种加载 Prompt 的隐式回退行为（除非文档明确允许）

### 2.3 条款映射

- [ ] `static/旅行险条款/<claim_type>/` 目录与文档一致
- [ ] `PolicyTermsRegistry` 映射项与文档一致
- [ ] 未配置 `claim_type` 时的行为与文档一致（应报错，不静默降级）

### 2.4 结论口径与故障分流

- [ ] 文档中的输出契约（`Remark/IsAdditional/KeyConclusions/DebugInfo`）与代码一致
- [ ] “系统异常 vs 证据不足”分流规则与代码一致
- [ ] 官方数据查询失败分类（如 `system_error`/`evidence_gap`）与文档一致

### 2.5 行李延误专项（baggage_delay）

- [ ] 时长核算主口径与文档一致（签收时间 - 首航实际到达）
- [ ] 多份签收时间取最晚规则已在代码实现
- [ ] 官方航班数据优先策略与文档一致
- [ ] 阶梯赔付规则与文档一致（6/12/18 小时）

## 3. ADR 对齐项

- [ ] ADR “落地范围”与当前代码状态一致
- [ ] ADR “验收标准”勾选状态与实际一致
- [ ] ADR “后续行动”中的已完成项已同步更新
- [ ] ADR 关联文档链接可用且指向正确文件
