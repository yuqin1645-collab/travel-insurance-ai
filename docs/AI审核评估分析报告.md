# AI审核评估分析报告

> 分析日期: 2026-04-27
> 数据范围: 390 个已审核案件，其中 39 个同时有人工处理状态
> 方法: 对比 AI 审核结论（本地 JSON）与人工最终状态（数据库 manual_status）

## 1. 本次案件总体误差来源

| 误差类型 | 数量 | 占比 | 主要层级 |

## 2. 字段级分析

### 字段1：has_baggage_tag_proof（行李牌有无）

- **状态：提取不稳定**
- **问题来源：AI字段提取层**
- **详情：** 同一类材料，AI 在不同案件中给出不同判断：
  - `a0nC800000HIKJTIA5`：vision 返回 `has_baggage_tag=false` → AI要求补件 → 人工通过（赔付1000元）
  - `a0nC800000HgiR3IAJ`：vision 返回 `has_baggage_tag=True` → 仍因其他材料要求补件 → 人工通过（赔付1000元）
  - `a0nC800000HWWKmIAP`：vision 返回 `has_baggage_tag=True`，AI 因时长不够拒赔 → 人工通过（赔付1500元）
- **根因：** 行李延误的"必备材料门禁"中，行李牌被设为必选项。但人工审核在实际操作中对该材料的宽容度较高（尤其是已有 PIR 报告 + 行李延误证明的情况下）。
- **建议：** 将行李牌从"必选"降为"加分项"，或与其他材料组合判断（有PIR+延误证明时可豁免行李牌）。

### 字段2：official_flight_data（官方航班数据）

- **状态：提取/验证逻辑过严**
- **问题来源：规则代码层 + Prompt约束**
- **详情：** 约 8/16 件"AI补件→人工通过"同时标记"官方航班数据未命中"。AI 要求有飞常准/官方数据才能通过，但人工在已有航空公司出具的行李延误证明时，不再要求额外官方数据。
- **根因：** 行李延误场景下，航空公司出具的行李延误证明本身就具有官方性质，但 AI 仍要求独立的第三方航班数据。
- **建议：** 在行李延误场景中，将"航空公司出具的行李延误/签收证明"视为等效官方数据。

### 字段3：delay_duration_minutes（航班延误时长）

- **状态：计算错误（高风险）**
- **问题来源：AI字段提取层 → 延误计算层**
- **详情：**
  - `a0nC800000MFr7NIAT`：AI 计算延误 8分钟 → 拒赔；人工通过（赔付600元）。案件描述写"延误接近12个小时"。**这是严重的计算错误**。
  - `a0nC800000M11a6IAB`：AI 计算延误 170分钟 → 拒赔（阈值300分钟）；人工通过（赔付600元）。案件描述提及航班取消+替代航班。AI可能用了原航班时刻而非替代航班时刻计算。
- **根因：** 航班延误时长计算逻辑在"改签/替代航班"场景下存在问题。AI 可能使用了错误的基准时间（原计划 vs 替代航班）。
- **建议：** 必须修复改签场景的延误计算逻辑，确保使用正确的基准时间。

### 字段4：flight_scenario（航班场景：直飞/联程/改签）

- **状态：识别存在歧义**
- **问题来源：AI语义判断层**
- **详情：**
  - `a0nC800000M6q8wIAB`：AI 识别为"中转延误"并适用除外责任（拒赔）；人工通过（赔付300元）。案件描述明确是替代航班（签转后航班），延误15小时。
  - `a0nC800000M1QQQIA3`：AI 识别为"纯国内航班"并拒赔；人工通过（赔付300元）。
- **根因：** AI 对"中转延误"和"纯国内航班"的除外责任判定过于机械，未考虑替代航班的实际场景。
- **建议：** 除外责任判定需结合"是否为保护航班/签转航班"综合判断。

### 字段5：policy_validity（保单有效期）

- **状态：AI遗漏**
- **问题来源：AI语义判断层**
- **详情：**
  - `a0nC800000Hl061IAB`：AI 仅要求补件（缺行李牌）；人工发现"出境后投保，保单自始无效"并拒绝。
- **根因：** AI 的材料门禁流程先于保单有效期检查，导致在补件阶段就返回，未触发保单有效期校验。
- **建议：** 保单有效期应作为第一优先级检查（Gate Check），在任何材料审核之前执行。

## 3. AI判断分析

### 3.1 行李延误"材料门禁"过度保守

在 16 件"AI补件→人工通过"中，**100% 是行李延误案件**。这些案件的共同模式：

```
AI流程：
  1. Vision 提取材料 → 发现缺行李牌照片 → 标记补件 → 直接返回
  2. 即使延误时长已达标（48h/144h）、有PIR报告、有延误证明
  3. 仍因单一材料缺失要求补件

人工实际流程：
  1. 有 PIR 报告 + 延误证明 + 签收时间 → 核心证据链完整 → 直接通过
  2. 行李牌照片作为辅助材料，非必须
```

**责任归属：Prompt约束过严 + 代码层材料门禁逻辑未区分"核心材料"和"辅助材料"。**

### 3.2 航班延误计算逻辑存在系统性偏差

4 件"AI拒赔→人工通过"的航班延误案件，全部涉及**延误时长计算偏差**或**除外责任误用**：

| 案件 | AI计算 | 实际描述 | AI判断 | 人工判断 | 差异原因 |
|------|--------|---------|--------|---------|---------|
| MFr7NIAT | 8分钟 | "延误接近12小时" | 拒赔 | 通过(600) | **计算错误** |
| M11a6IAB | 170分钟 | 航班取消+替代 | 拒赔 | 通过(600) | **基准时间选错** |
| M1QQQIA3 | 300分钟达标 | 杭州→上海国内 | 拒赔(纯国内) | 通过(300) | **除外责任适用争议** |
| M6q8wIAB | 900分钟达标 | 签转替代航班 | 拒赔(中转) | 通过(300) | **除外责任误用** |

**责任归属：AI语义判断层 + 延误计算代码层。**

### 3.3 赔付金额为0的逻辑问题

3 件行李延误案件（HbxYpIAJ, HcbG5IAJ, HepdCIAR）AI输出"审核通过，建议赔付0.00元"，但人工实际赔付1000元。

**根因：** AI 虽然通过了审核，但赔付档位计算可能未正确触发（可能是 delay_hours 类型/格式问题导致档位查找失败）。

## 4. 代码覆盖分析

### 4.1 无直接代码覆盖

对比 AI 原始 JSON 输出与数据库最终值，**未发现代码覆盖导致结论反转的情况**。数据库的 `audit_result`、`payout_amount` 与 AI JSON 文件一致。

### 4.2 间接影响：`final_decision` 字段全空

数据库中 1087 条记录的 `final_decision` 字段全部为空。这意味着：
- `_extract_review_fields` 从 JSON 提取 `final_decision` 时，AI 输出的该字段本身可能为空
- 或者提取逻辑未正确映射

**影响：** 无法通过数据库直接查询 AI 的最终决定（通过/拒赔/补件），只能通过解析 JSON 文件或 `audit_result` 字段推断。

### 4.3 赔付金额差异（非覆盖导致）

`a0nC800000MHnEDIA1`：AI 输出"延误7小时，赔付0元"，人工通过但数据库 payout_amount 为 600元。
- 这不是代码覆盖，而是 AI 计算为 0 后，人工审核覆盖了金额。
- 数据库写入的是人工推送回来的值（通过前端接口回写）。

## 6. 共性问题累计统计

### 共性问题 TOP5

1. **行李延误材料门禁过严（16件，占不一致案例64%）**
   - AI 要求"托运行李牌照片"作为必选项
   - 人工在有 PIR + 延误证明 + 签收时间时直接通过
   - **属于：Prompt问题 + 代码问题**（材料分级逻辑缺失）

2. **改签/替代航班延误计算错误（2件，严重）**
   - AI 使用了错误的基准时间计算延误时长
   - `MFr7NIAT`：描述"延误12小时"，AI算8分钟
   - `M11a6IAB`：航班取消后替代，AI算170分钟而非完整延误
   - **属于：代码问题**（延误计算逻辑缺陷）

3. **除外责任适用过于机械（2件）**
   - "纯国内航班"和"中转延误"除外责任在替代航班场景下被误用
   - 人工在替代航班/签转场景下仍然赔付
   - **属于：AI判断问题 + Prompt问题**（未充分定义例外情况）

4. **保单有效期检查优先级不足（1件）**
   - AI 先进入材料审核流程，未首先校验保单有效期
   - 人工发现"出境后投保，保单自始无效"并拒绝
   - **属于：代码问题**（审核流程顺序缺陷）

5. **赔付金额计算为0的异常（3件）**
   - AI 通过审核但赔付金额为 0，与实际档位不符
   - **属于：代码问题**（档位查找函数输入/输出异常）

## 7. 下一步优先优化建议

### P0 - 立即修复

| 优化项 | 影响范围 | 预期效果 |
|--------|---------|---------|
| **修复改签场景延误计算逻辑** | 航班延误核心功能 | 消除 8分钟 vs 12小时 类严重错误 |
| **行李延误材料分级**：将材料分为"核心证据链"（延误证明+签收时间+PIR）和"辅助材料"（行李牌照片、登机牌）。核心材料齐全即可通过 | 16件不一致案例中占13件 | 减少约 80% 的"AI补件→人工通过"差异 |

### P1 - 短期优化

| 优化项 | 影响范围 | 预期效果 |
|--------|---------|---------|
| **保单有效期前置为 Gate Check**：在任何材料审核前先校验保单有效期 | 所有险种 | 避免遗漏"保单自始无效"的拒赔案件 |
| **除外责任增加"替代航班例外"**：签转/保护航班不适用纯国内/中转延误除外 | 航班延误 | 消除机械适用除外责任的问题 |
| **赔付档位查找增加日志和兜底**：记录档位查找的输入值，异常时给出警告 | 行李延误 | 消除"赔付0元"异常 |

### P2 - 中期优化

| 优化项 | 影响范围 | 预期效果 |
|--------|---------|---------|
| **official_flight_data 等效认定**：行李延误场景下，航空公司出具的延误/签收证明视为等效官方数据 | 行李延误 | 减少约 50% 的"官方数据未命中"补件 |
| **`final_decision` 字段写入修复**：确保 AI 审核结论正确写入数据库 | 全部 | 支持数据库直接查询和统计分析 |
| **引入"AI-人工差异监控"**：定期对比 AI 结论与人工结论，自动标记差异案件 | 全部 | 持续追踪 AI 审核准确率 |
