### 状态值统一：需补件 → 需补齐资料

**8件异常 flight_delay `需补件`**：重跑后全部修复（其中1件为代码崩溃 `name 'ctx' is not defined`，其余7件为旧版代码遗留的状态值）。

### 核心修复逻辑

**联程航班延误计算规则（与人工对齐）：**

1. **只看末段**：联程原航班（如 CA961 PEK→MUC + LH2440 MUC→CPH），延误计算以**末段**（MUC→CPH）的计划到达时间为基准，不是首段（PEK→MUC）的计划出发时间
2. **机场匹配**：替代航班（如 CA877 PEK→CPH 直飞）的 dep/arr iata 必须与末段机场匹配，dep 不匹配（MUC≠PEK）时起飞延误分量不计算
3. **前程延误判定**：若前程正常到达中转机场（飞常准确认），末段独立取消/延误不触发「前程延误导致误机」免责
4. **单段不误标**：Vision 标记 `is_connecting_rebooking` 前校验 `itinerary_segments` 数量，单段直飞不标记为联程改签
